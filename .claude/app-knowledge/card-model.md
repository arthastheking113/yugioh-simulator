# Card Model

## Card Properties

| Property | Type | Values | Description |
|----------|------|--------|-------------|
| `cardId` | Number | e.g. `48130397` | Card database ID |
| `uuid` | String | custom (not RFC-4122) | Unique instance ID — auto-generated by `ygoUUID()`; template `'xyxy-xxyy-0510-xyyy-xxxx'` |
| `name` | String | card name | Display name |
| `position` | String | see Positions | Current zone |
| `foldState` | String | `normal`, `fold` | Face-up / face-down |
| `switchState` | String | `attack`, `defense` | Battle position |
| `collection_order` | Number or String | `"ss3"` / `7` | Slot **token** in individual zones (`ss1`–`ss5`, `exss1`/`exss2`, `st1`–`st5`, `fz1`); numeric stack index in collections |
| `isSpell` | Boolean | | Spell card |
| `isTrap` | Boolean | | Trap card |
| `isMonster` | Boolean | | Monster card |
| `isST` | Boolean | | Spell or Trap (either) |
| `isExtra` | Boolean | | Extra deck card |
| `imageURL` | String | URL or `''` | Card art; fallback: `asset/card/{cardId}.jpeg` |
| `description` | String | card text | Shown on hover in the info panel |
| `isOverlap` | Boolean | | True if this card IS an Xyz material (sits beneath the monster) |
| `isOverlay` | Boolean | | True if this card IS the Xyz monster carrying materials (on top) |
| `overlap_order` | Number | 0, 1, 2, 3… | Stacking position of this material (0 = not a material) |

## Positions (the `position` field)

| Value | Zone | Slot count |
|-------|------|-----------|
| `hand` | Player hand | Unlimited |
| `deck` | Main deck (collection) | Unlimited |
| `exdeck` | Extra deck (collection) | Unlimited |
| `summon` | Monster zones | 5 (slots 1–5) |
| `st` | Spell/trap zones | 5 (slots 1–5) |
| `fz` | Field zone | 1 |
| `graveyard` | Graveyard (collection) | Unlimited |
| `banish` | Banished zone (collection) | Unlimited |

## CSS Classes Derived from Card State

Verified against `Card.drawHtml()` / `updateHtml()` and `css/simulator.css` — the element is `.simulator-card`, and state classes are the **bare property values** (no `.card-item`, `.fold-state-*`, `.position-*`, `.is-*`, `.card-targeted`, `.card-declared` — those names do not exist):

```
.simulator-card                     always present (base class); also .card-id-{uuid}
.normal | .fold                     foldState value (face-up / face-down)
.attack | .defense                  switchState value (.defense rotates 90°)
.hand .deck .exdeck .summon .st .fz .graveyard .banish   position value
.isMonster | .isSpell | .isTrap | .isST   card type flags (literal names, added when flag == 1)
.overlap                            Xyz material (this card IS a material; isOverlap === true)
.overlay                            Xyz monster carrying materials (isOverlay === true)
```

Target/declare/reveal/phase animations use Animate.css classes (`animate__animated animate__zoomIn`, `animate__fadeOut`, …) injected by `doAnimation()`, not dedicated state classes. The holder slot gets `.overlay-slot` when it holds an Xyz pile.

## Card Type Flags

A card sets exactly one of `isMonster`, `isSpell`, `isTrap`. `isST` is `true` when `isSpell || isTrap`. `isExtra` is `true` for Fusion, Synchro, Xyz, Link monsters.

## UUID Generation

`ygoUUID()` uses `Date.now()` + `performance.now()` microseconds. It is **not** a standard UUID v4 — the actual template string is `'xyxy-xxyy-0510-xyyy-xxxx'` (note the literal `0510` segment). Auto-assigned in the `Card` constructor (and defensively in `board.addItem()`) when `uuid` is `0`.

## Image Loading

1. Use `card.imageURL` if provided (CDN: `ygovietnamcdn.azureedge.net`)
2. Fallback: `asset/card/{cardId}.jpeg`
